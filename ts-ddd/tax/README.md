# Tax Application

TypeScriptによるClean Architecture likeな実装

### infra
インフラ層。
クリーンアーキテクチャ的にはFrameworks and Drivers層という名称がついている。
システムの受け口でもあり、処理結果を表示したり、出力する層でもある。
- CLIやWEBなどからの入力を受け付ける層
- RDBなどへの具体的な永続化処理などの具体的な実装もここで書く

### interface
インターフェースアダプター層。
MVCでいうならコントローラーが置かれたりなどする。
- 「ユースケースやエンティティに便利なフォーマットから，DBやWEBなどの外部エージェントに便利なフォーマットにデータを変換する」などを担う
- つまりはUseCase(Application)やEntity(Domain)とDrivers(Infra)間でのデーターの変換処理などを担う

### app
アプリケーション層を指す。クリーンアーキテクチャ的にはUseCaseの事。
- 「アプリケーション固有のビジネスルール」をここで実装する
- Domain層にあるEntityをカプセル化し、ビジネスルールを実行
- 「データを保存する」といった事はこの辺りが担う

### domain
ドメイン層。
クリーンアーキテクチャ的にはEntityなどを置く。
- 「最重要のビジネスルールをカプセル化したもの」をここに置く
